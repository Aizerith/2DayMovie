## Ajouter une langue
1. Dupliquer `fr.ts` ou `en.ts` en `es.ts`, `de.ts`, etc.
2. Enregistrer la langue dans [transloco.providers.ts](C:\Users\skyry\IdeaProjects\BoilerPlate-Spring-Angular\frontend\src\app\core\i18n\transloco.providers.ts) via `availableLangs`.
3. Ajouter le chargement dans [transloco-loader.ts](C:\Users\skyry\IdeaProjects\BoilerPlate-Spring-Angular\frontend\src\app\core\i18n\transloco-loader.ts).
4. Si besoin, ajouter l option dans le switch de langue des layouts.

## Ajouter une cle
- privilegier un namespace par feature : `home.*`, `login.*`, `projects.*`, `tasks.*`, `users.*`
- reserver `common.*` aux libelles transverses
- reserver `validation.*` aux messages de formulaire
- reserver `enums.*` aux labels derives des statuts/roles/priorites

## Bonnes pratiques
- utiliser le pipe `transloco` dans les templates
- utiliser [i18n.service.ts](C:\Users\skyry\IdeaProjects\BoilerPlate-Spring-Angular\frontend\src\app\core\i18n\i18n.service.ts) pour les messages imperatifs en TypeScript
- eviter de laisser des labels visibles en dur dans les templates
- garder les placeholders metier dans les fichiers de traduction quand ils ont un vrai sens produit

## Validation
- build frontend : `ng build`
- tests front : `ng test --watch=false`
- verifier a la main le switch `FR / EN` sur les layouts public et prive
